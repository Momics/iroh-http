# [ISS-015] Sign/verify docs claim synchronous API; implementation is async

## Summary

The sign/verify feature documentation states "Both sign and verify are synchronous," but the implementation uses async WebCrypto operations.

## Evidence

- `docs/features/sign-verify.md:20` — states sign and verify are synchronous
- `packages/iroh-http-shared/src/keys.ts:98` — `sign` uses async WebCrypto
- `packages/iroh-http-shared/src/keys.ts:250` — `verify` uses async WebCrypto

## Impact

Callers following the documentation will misuse the API by not awaiting the returned promises.

## Remediation

1. Update documentation and all examples to use `await` semantics for sign and verify.

## Acceptance criteria

1. All sign/verify documentation and examples compile and run as written using async/await.
