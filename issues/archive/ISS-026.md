# [ISS-026] Release automation workflow missing

## Summary

The roadmap calls for a tag-triggered `release.yml` workflow that publishes artifacts to npm, JSR, PyPI, and crates.io. Only a CI workflow exists today.

## Evidence

- `docs/roadmap.md:39` — calls out missing `release.yml` as a blocker
- `.github/workflows/ci.yml:1` — only CI workflow currently exists

## Impact

Releases must be performed manually, introducing risk of inconsistent artifact publishing and human error.

## Remediation

1. Add a `v*` tag-triggered release workflow that builds and publishes to npm/JSR, PyPI, and crates.io as planned.

## Acceptance criteria

1. A `v*` tag triggers the pipeline and successfully publishes release artifacts to all registries.
