# [ISS-016] Protocol doc references wrong duplex client API shape

## Summary

The protocol documentation shows `node.createBidirectionalStream()` as the duplex API. The actual API requires a session object: `node.connect(peer)` returns `IrohSession`, then `session.createBidirectionalStream()`.

## Evidence

- `docs/protocol.md:80` — shows `node.createBidirectionalStream()`
- `packages/iroh-http-shared/src/bridge.ts:493` — actual duplex method lives on `IrohSession`
- `packages/iroh-http-shared/src/session.ts:160` — `createBidirectionalStream()` on session object

## Impact

Developers reading the protocol docs will write code against a non-existent API and receive runtime errors.

## Remediation

1. Update the protocol documentation to show the correct session-based API flow.

## Acceptance criteria

1. Protocol documentation examples mirror the current public API.
