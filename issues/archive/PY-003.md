# [PY-003] `HandlerResponse` not re-exported from package root

## Summary

`HandlerResponse` is registered as a native PyO3 class, but it is omitted from `iroh_http/__init__.py`. Users who follow the documented API get an `ImportError` at runtime.

## Evidence

- `packages/iroh-http-py/src/lib.rs:1028` тАФ `HandlerResponse` registered as native class
- `packages/iroh-http-py/iroh_http/__init__.py:29` тАФ not included in re-exports
- `packages/iroh-http-py/tests/test_node.py:185` тАФ test fails with `ImportError`

## Impact

The documented `HandlerResponse` API is completely broken for users.

## Remediation

1. Add `HandlerResponse` to the re-exports in `iroh_http/__init__.py`.
2. Add or adjust a stub entry for `HandlerResponse` in `__init__.pyi`.

## Acceptance criteria

1. `from iroh_http import HandlerResponse` succeeds.
2. Type stubs reflect the `HandlerResponse` class.
