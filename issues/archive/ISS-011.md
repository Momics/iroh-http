# [ISS-011] Streaming docs claim FormData support; code rejects FormData

## Summary

The streaming feature documentation lists `FormData` as a supported `BodyInit` type, but the shared implementation throws when a `FormData` value is passed.

## Evidence

- `docs/features/streaming.md:28` — docs list `FormData` as a supported `BodyInit`
- `packages/iroh-http-shared/src/streams.ts:104` — implementation throws on `FormData`

## Impact

Users following the documentation who pass `FormData` as a request body will receive a runtime error.

## Remediation

1. Either implement `FormData` serialization in the body handling path, or remove it from the docs as an unsupported type.

## Acceptance criteria

1. `FormData` behavior is explicit, consistent, and tested across docs, types, and runtime.
