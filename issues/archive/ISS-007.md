# [ISS-007] Request timeout returns generic error instead of HTTP 408

## Summary

When a request times out, the server returns a generic boxed error string. Documentation promises an HTTP `408 Request Timeout` response.

## Evidence

- `crates/iroh-http-core/src/server.rs:638` — timeout service returns a generic boxed error string
- `docs/features/server-limits.md:50` — docs claim timeout behavior maps to `408`

## Impact

Timeout status mapping is inconsistent across runtimes and adapters, making it impossible for callers to reliably detect a timeout via HTTP status code.

## Remediation

1. Map the timeout error to an explicit HTTP `408 Request Timeout` response path.
2. Ensure adapters preserve the status code semantics through the FFI boundary.

## Acceptance criteria

1. A request that times out returns `408` consistently across all adapters.
