# [PY-007] Invalid `direct_addrs` are silently dropped in `connect` and `fetch`

## Summary

In both `connect` and `fetch`, `direct_addrs` is parsed with `filter_map(...ok())`, silently discarding any addresses that fail to parse. Address typos produce no error and cause confusing fallback behavior.

## Evidence

- `packages/iroh-http-py/src/lib.rs:557` — `filter_map(...ok())` in `connect`
- `packages/iroh-http-py/src/lib.rs:605` — `filter_map(...ok())` in `fetch`

## Impact

A misconfigured or typo'd address is silently ignored, potentially sending traffic to the wrong peer or failing to connect without a useful error message.

## Remediation

1. Fail fast when any provided address string is invalid.
2. Include the offending address string in the error message.

## Acceptance criteria

1. Passing an invalid address string in `direct_addrs` raises a Python exception with the offending value.
