# [PY-001] `serve()` can panic due to missing Tokio reactor context

## Summary

The Python `serve` method is synchronous but immediately enters an async/Tokio-dependent serving path. Calling it outside a Tokio runtime panics with "there is no reactor running", which PyO3 surfaces as `pyo3_runtime.PanicException`.

## Evidence

- `packages/iroh-http-py/src/lib.rs:637` — `serve` enters Tokio-dependent path synchronously
- `packages/iroh-http-py/src/lib.rs:645` — deeper async serving logic
- `packages/iroh-http-py/tests/test_node.py:79` — test failures show the panic

## Impact

User code calling `node.serve(handler)` in a normal pytest or synchronous context unconditionally raises `pyo3_runtime.PanicException`.

## Remediation

1. Ensure `serve` executes within a Tokio runtime context (e.g., enter the existing runtime or create one if absent).
2. Add a regression test that calls `serve` in the standard pytest-asyncio execution mode.

## Acceptance criteria

1. `node.serve(handler)` in a pytest test does not raise `PanicException`.
