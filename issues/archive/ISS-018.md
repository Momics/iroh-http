# [ISS-018] Internal comments reference retired QPACK/v1 details

## Summary

Several inline comments in the endpoint module reference old ALPN capability sets and QPACK-style language from an earlier protocol version that has since been replaced by v2/hyper.

## Evidence

- `crates/iroh-http-core/src/endpoint.rs:40` — comment references old ALPN capability set
- `crates/iroh-http-core/src/endpoint.rs:73` — QPACK language in comment
- `crates/iroh-http-core/src/lib.rs:182` — current protocol constants are v2

## Impact

Maintainers are given incorrect design context, increasing the risk of implementation mistakes and protocol intent drift.

## Remediation

1. Update stale inline comments to reflect the current v2/hyper transport model.

## Acceptance criteria

1. Inline docs in `endpoint.rs` match current transport and wire behavior.
