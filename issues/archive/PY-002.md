# [PY-002] Unsafe raw-pointer lifetime in `IrohBrowseSession.__anext__`

## Summary

`__anext__` converts `self` to a `usize` raw pointer and captures it inside an async future. If the Python object is garbage-collected before the future completes, the pointer is dangling, causing use-after-free or undefined behavior.

## Evidence

- `packages/iroh-http-py/src/lib.rs:324` — `self` cast to raw pointer
- `packages/iroh-http-py/src/lib.rs:327` — raw pointer dereferenced inside async future

## Impact

Any cancellation or GC of the browse session during async iteration leads to undefined behavior with no safety net.

## Remediation

1. Replace the raw-pointer capture with an owned `Py<...>` or `Arc`-backed state that guarantees lifetime across the await point.
2. Add cancellation and drop tests for browse iteration.

## Acceptance criteria

1. Dropping `IrohBrowseSession` mid-iteration does not cause a crash or undefined behavior.
