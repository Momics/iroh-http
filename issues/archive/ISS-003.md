# [ISS-003] Server-side `maxHeaderBytes` contract is incomplete

## Summary

The server only sets a hyper parser limit for header size. There is no post-parse byte counting or explicit `431 Request Header Fields Too Large` response path, both of which are required by internal docs and feature documentation.

## Evidence

- `crates/iroh-http-core/src/server.rs:573` — only parser limit is set; no post-parse enforcement found
- `docs/internals/http-engine.md:158` — enforcement must be: parser clamp + post-parse byte count
- `docs/features/server-limits.md:52` — feature docs promise `431` behavior

## Impact

Behavioral mismatch between documented and actual header-limit semantics. Callers expecting a `431` response receive inconsistent behavior.

## Remediation

1. Compute measured request header bytes server-side after parse.
2. Return `431 Request Header Fields Too Large` when the measured size exceeds the configured limit.
3. Surface a structured `HEADER_TOO_LARGE` error code path.

## Acceptance criteria

1. Requests with headers above the configured limit receive a `431` response.
2. `ErrorCode::HeaderTooLarge` is reachable from a real code path.
