# [ISS-010] Server-limit docs describe `serve()` options; limits live at `createNode()`

## Summary

The server-limits documentation shows limits configured on `node.serve({ maxConcurrency, ... }, handler)`, but the actual implementation configures them at `createNode(...)`. The `ServeOptions` type contains no limit fields.

## Evidence

- `docs/features/server-limits.md:10` — docs show `node.serve({ maxConcurrency, ... }, handler)`
- `packages/iroh-http-shared/src/serve.ts:40` — `ServeOptions` only contains lifecycle/error hooks
- `packages/iroh-http-shared/src/serve.ts:159` — `makeServe` passes empty options to raw layer
- `packages/iroh-http-node/lib.ts:374` — limits are provided at endpoint creation
- `packages/iroh-http-deno/src/adapter.ts:447` — same pattern in Deno adapter

## Impact

Developers following the docs will pass limits to `serve()` where they are silently ignored, then see unexpected behavior.

## Remediation

1. Update docs to the `createNode({ ...limits })` model, or implement serve-time override.
2. If a serve-time override is added, clearly define precedence rules.

## Acceptance criteria

1. Public docs and type signatures match one consistent configuration model.
