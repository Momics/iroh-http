# [ISS-027] `close()` unregisters endpoint handles before graceful drain

## Summary

`IrohEndpoint::close()` removes all endpoint-owned slab entries before draining the serve handle. This inverts graceful-shutdown ordering and can invalidate in-flight request state during the drain window.

## Evidence

- `crates/iroh-http-core/src/endpoint.rs:332` — `close()` calls `unregister_endpoint` first
- `crates/iroh-http-core/src/endpoint.rs:341` — drain happens after unregister
- `crates/iroh-http-core/src/endpoint.rs:343` — endpoint close happens after drain

## Impact

Requests that should still be draining can lose reader/writer/trailer/request-head state prematurely, producing spurious internal errors and violating graceful shutdown semantics.

## Remediation

1. Reorder graceful shutdown to drain first, then unregister endpoint-owned slab entries, then close the endpoint.
2. Keep force-close behavior explicit and separate from graceful-close semantics.

## Acceptance criteria

1. During graceful `close()`, in-flight requests can finish without slab-handle invalidation errors.
2. Endpoint-owned slab entries are only removed after drain completes (or times out).
