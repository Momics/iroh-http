# [PY-014] Type stubs omit path_changes and IrohPathChanges runtime API

## Summary

The Python runtime exports `node.path_changes(...)` and `IrohPathChanges`, but the `.pyi` stubs do not declare this API. This creates typing drift and hides supported functionality from IDEs and static checkers.

## Evidence

- `packages/iroh-http-py/src/lib.rs:548` — runtime class `IrohPathChanges` is defined.
- `packages/iroh-http-py/src/lib.rs:871` — `IrohNode.path_changes(...)` is implemented.
- `packages/iroh-http-py/src/lib.rs:1238` — module exports `IrohPathChanges`.
- `packages/iroh-http-py/iroh_http/__init__.pyi:137-194` — stubs define `IrohNode` methods but omit `path_changes` and `IrohPathChanges`.

## Impact

Typed Python users get incomplete auto-complete and false negatives for valid runtime API usage.

## Remediation

1. Add `IrohPathChanges` class declaration to `__init__.pyi` with async iterator signatures.
2. Add `IrohNode.path_changes(node_id: str) -> IrohPathChanges` to the stub.
3. Ensure top-level exports and stubs remain synchronized when new public APIs are added.

## Acceptance criteria

1. `__init__.pyi` contains `IrohPathChanges` and `IrohNode.path_changes`.
2. Type checkers accept valid `async for event in node.path_changes(peer_id)` usage.
3. Stub/API parity checks (manual or automated) cover this surface going forward.
