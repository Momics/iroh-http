# [ISS-032] Deno FFI JSON bridge downcasts `bigint` handles to `number`

## Summary

The Deno adapter serializes `bigint` values by converting them to `number` at the JSON boundary. Handle values are currently expected to stay small, but the bridge assumes precision safety for all `u64` values.

## Evidence

- `packages/iroh-http-deno/src/adapter.ts:124` — comment claims slotmap handle safety within f64 range
- `packages/iroh-http-deno/src/adapter.ts:126` — `typeof v === "bigint" ? Number(v) : v`

## Impact

If handle values ever exceed JS safe integer range, precision loss can corrupt handle identity and route operations to wrong resources. Even if unlikely today, the boundary contract is unsound for full `u64`.

## Remediation

1. Use a lossless JSON encoding for bigint handles (e.g., decimal strings) and parse back explicitly.
2. Document and enforce any handle-range assumptions if retaining numeric transport.

## Acceptance criteria

1. Deno bridge round-trips handle identifiers without precision loss.
2. Large-handle behavior is either safe by construction or rejected with a clear error.
