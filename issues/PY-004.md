# [PY-004] `__all__` exports `IrohBrowseSession` unconditionally

## Summary

`iroh_http/__init__.py` conditionally imports `IrohBrowseSession` (swallowing the `ImportError` if the `mdns` feature is absent), but always includes `IrohBrowseSession` in `__all__`. This causes `from iroh_http import *` to fail with `AttributeError` on non-`mdns` builds.

## Evidence

- `packages/iroh-http-py/iroh_http/__init__.py:42` — conditional import swallows absence
- `packages/iroh-http-py/iroh_http/__init__.py:58` — `__all__` always includes `IrohBrowseSession`

## Impact

Wildcard imports from `iroh_http` fail on any build that does not include the `mdns` feature flag.

## Remediation

1. Only append `IrohBrowseSession` to `__all__` when the import succeeds.
2. Add a test that verifies `from iroh_http import *` works on a non-`mdns` build.

## Acceptance criteria

1. `from iroh_http import *` succeeds on a non-`mdns` build without `AttributeError`.
