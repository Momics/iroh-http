# [ISS-033] Pre-release cleanup: remove deprecated API shims and legacy error fallbacks

## Summary

The repository still carries deprecated API aliases and compatibility shims across shared and adapter layers, plus legacy free-form error classification fallbacks. Since the library is pre-release, these paths can be removed now to simplify the public surface and error model.

## Evidence

- `packages/iroh-http-shared/src/bridge.ts:221` — deprecated `NodeOptions.dnsDiscovery` retained
- `packages/iroh-http-shared/src/bridge.ts:365` — deprecated `NodeOptions.lifecycle` retained
- `packages/iroh-http-shared/src/bridge.ts:439` — deprecated `IrohNode.nodeId` retained
- `packages/iroh-http-shared/src/bridge.ts:441` — deprecated `IrohNode.keypair` retained
- `packages/iroh-http-shared/src/bridge.ts:692` — deprecated `DuplexStream` alias retained
- `packages/iroh-http-node/lib.ts:345` — `dnsDiscovery` compatibility fallback still wired
- `packages/iroh-http-deno/src/adapter.ts:430` — `dnsDiscovery` compatibility fallback still wired
- `packages/iroh-http-tauri/guest-js/index.ts:539` — `lifecycle` compatibility fallback still wired
- `packages/iroh-http-shared/src/index.ts:170` — runtime object still exposes deprecated `nodeId`
- `packages/iroh-http-shared/src/index.ts:171` — runtime object still exposes deprecated `keypair`
- `crates/iroh-http-core/src/lib.rs:145` — legacy `classify_error_json` preserved for adapter compatibility
- `packages/iroh-http-shared/src/errors.ts:221` — legacy regex fallback classifier still active
- `packages/iroh-http-node/test/smoke.mjs:16` — tests still validate deprecated `node.nodeId`
- `packages/iroh-http-deno/test/smoke.test.ts:31` — tests still validate deprecated `node.keypair`
- `packages/iroh-http-tauri/README.md:41` — docs still instruct deprecated `lifecycle` option

## Impact

Deprecated paths increase maintenance and create two parallel API contracts (new and old) before first release. Legacy string/regex error fallbacks weaken the structured error-code contract and can hide classification drift behind permissive matching.

## Remediation

1. Remove deprecated fields/types from shared public types (`dnsDiscovery`, `lifecycle`, `nodeId`, `keypair`, `DuplexStream`).
2. Remove adapter fallback plumbing that maps deprecated inputs to current options.
3. Remove runtime emission of deprecated aliases from `buildNode`.
4. Complete migration to structured error mapping (`core_error_to_json` + code-based classification) and delete regex fallback paths.
5. Update tests and docs to assert only canonical API (`publicKey`, `secretKey`, `reconnect`, structured error codes).

## Acceptance criteria

1. No deprecated API entries remain in `packages/iroh-http-shared/src/bridge.ts`.
2. Node/Deno/Tauri adapters no longer read `dnsDiscovery` or `lifecycle` compatibility fields.
3. `buildNode` no longer returns deprecated `nodeId`/`keypair` properties.
4. `classify_error_json` and JS regex fallback classification are removed, with structured code paths used end-to-end.
5. Existing tests/docs are updated to the canonical API and pass without deprecated aliases.
