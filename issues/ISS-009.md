# [ISS-009] Error enum has drifted/unused variants

## Summary

`HeaderTooLarge` and `PeerRejected` are declared in the error enum but are not referenced in any active logic path, creating taxonomy ambiguity between docs and behavior.

## Evidence

- `crates/iroh-http-core/src/lib.rs:46` — variants declared in the enum
- Neither variant is referenced in active logic paths beyond status mapping

## Impact

The error enum advertises capabilities that don't exist at runtime. Docs and behavior can diverge silently as new paths are added.

## Remediation

1. Either implement these variants in real code paths, or remove/deprecate them.
2. Keep the enum and status mapping aligned with actual server behavior.

## Acceptance criteria

1. Every public error code has at least one concrete origin path in the codebase.
