# [ISS-019] Compression `level` option accepted everywhere but ignored in core

## Summary

All adapters and documentation expose a configurable `compression.level` option, but `CompressionOptions` in core only has `min_body_bytes`. The level value is carried through the FFI but silently dropped.

## Evidence

- `docs/features/compression.md:15` — advertises configurable `compression.level`
- `packages/iroh-http-node/lib.ts:366` — passes `compressionLevel`
- `packages/iroh-http-deno/src/adapter.ts:437` — passes `compressionLevel`
- `packages/iroh-http-node/src/lib.rs:93` — FFI carries `compression_level`
- `packages/iroh-http-deno/src/dispatch.rs:150` — FFI carries `compression_level`
- `crates/iroh-http-core/src/endpoint.rs:101` — `CompressionOptions` has no `level` field

## Impact

The compression level setting is a silent no-op. Callers tuning compression for performance get no effect.

## Remediation

1. Either implement zstd level wiring end-to-end in core, or remove `level` from public options, docs, and all adapter types.

## Acceptance criteria

1. `compression.level` either works and has a test, or is not exposed in any public API.
