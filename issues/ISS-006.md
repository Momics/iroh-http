# [ISS-006] `maxConcurrency` docs claim 503; code queues and waits

## Summary

When the server reaches `maxConcurrency`, requests block on a semaphore waiting for a slot to free up. Documentation promises a fast-fail `503 Service Unavailable` response.

## Evidence

- `crates/iroh-http-core/src/server.rs:544` — semaphore `.acquire()` blocks; no timeout or rejection path
- `docs/features/server-limits.md:48` — docs claim overload returns `503`

## Impact

Under high load, excess requests queue silently and experience unbounded latency instead of receiving the documented `503` failure response.

## Remediation

1. Either implement fast-fail `503` when no permit is immediately available, or update docs to reflect queue semantics.
2. If queue semantics are kept, document timeout interactions for queued requests.

## Acceptance criteria

1. Over-capacity behavior is deterministic, and docs accurately describe it.
