# [ISS-013] `NodeOptions.injectHeaders` referenced in docs but does not exist

## Summary

The default-headers feature documentation mentions an optional `NodeOptions.injectHeaders` field for injecting `iroh-relay` and `iroh-rtt-ms` headers, but this field does not exist in `NodeOptions`.

## Evidence

- `docs/features/default-headers.md:34` — documents optional `NodeOptions.injectHeaders`
- `packages/iroh-http-shared/src/bridge.ts:154` — `NodeOptions` has no `injectHeaders` field

## Impact

The feature appears available but cannot be used, leading to silent no-op behavior for developers who try it.

## Remediation

1. Either implement `injectHeaders` end-to-end, or remove the reference from the docs.

## Acceptance criteria

1. The option either exists with tested behavior, or docs no longer advertise it.
