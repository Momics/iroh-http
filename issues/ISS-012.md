# [ISS-012] Observability docs describe APIs and fields not implemented

## Summary

The observability documentation advertises a `stats(): Promise<NodeStats>` method and a richer `PeerStats` shape (including `rtt`, `bytes`, `pathSelected`). Neither exists in the actual shared types or Rust core.

## Evidence

- `docs/features/observability.md:11` — advertises `stats()` and extended `PeerStats` fields
- `packages/iroh-http-shared/src/bridge.ts:619` — actual `PeerStats` shape is smaller
- `packages/iroh-http-shared/src/index.ts:282` — `stats()` is not exported
- `crates/iroh-http-core/src/endpoint.rs:455` — Rust endpoint has no `stats()` method

## Impact

Consumers cannot rely on the documented telemetry API; any code written against the documented shape will fail at compile time or runtime.

## Remediation

1. Either implement the missing observability API surface, or narrow docs to the current shape.

## Acceptance criteria

1. Type definitions, docs, and runtime for observability are fully aligned.
