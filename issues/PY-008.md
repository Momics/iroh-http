# [PY-008] Type stubs do not match runtime behavior

## Summary

The `.pyi` stub file declares `close()` as synchronous, but the runtime implementation returns an awaitable. `HandlerResponse` is also absent from the stubs. Downstream type checkers and IDEs give incorrect guidance.

## Evidence

- `packages/iroh-http-py/iroh_http/__init__.pyi:93` — stub declares `close()` as sync
- `packages/iroh-http-py/src/lib.rs:410` — runtime exposes awaitable `close`
- `HandlerResponse` is missing from stubs (also tracked in PY-003)

## Impact

Type checkers and IDE auto-complete give incorrect guidance, causing runtime errors in code that follows stub hints.

## Remediation

1. Update `.pyi` stubs to reflect awaitable return types for all PyO3 async methods.
2. Add `HandlerResponse` to the stubs.
3. Add conditional mDNS typing notes for `IrohBrowseSession`.

## Acceptance criteria

1. Running `mypy` or `pyright` against well-formed usage of the async methods produces no false errors.
