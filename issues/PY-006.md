# [PY-006] `next_unidirectional_stream` returns `IrohBidiStream` with invalid write handle

## Summary

`next_unidirectional_stream` returns an `IrohBidiStream` with `write_handle: 0`, exposing a writable API surface on a receive-only stream. Write or close calls on this object invoke undefined or error-prone behavior.

## Evidence

- `packages/iroh-http-py/src/lib.rs:484` — returns `IrohBidiStream` for a unidirectional stream
- `packages/iroh-http-py/src/lib.rs:486` — `write_handle` set to `0`

## Impact

Python code that receives a unidirectional stream and attempts to use it reasonably sees undefined behavior or panics when writing through the invalid handle.

## Remediation

1. Return a dedicated receive-only stream type from `next_unidirectional_stream`.
2. At minimum, guard write and close operations on handle `0` with a clear Python exception.

## Acceptance criteria

1. Calling `write()` on a stream returned by `next_unidirectional_stream` raises a descriptive exception.
