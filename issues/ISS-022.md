# [ISS-022] Test suite uses sleep-based timing despite principle prohibiting it

## Summary

Multiple tests across Rust and JS use `tokio::time::sleep` / `setTimeout` for synchronization. The codebase principles explicitly prohibit sleep-based test timing.

## Evidence

- `crates/iroh-http-core/tests/integration.rs:625` — sleep-based synchronization
- `crates/iroh-http-core/tests/bidi_stream.rs:159` — sleep-based synchronization
- `packages/iroh-http-node/test/e2e.mjs:157` — `setTimeout` in test
- `packages/iroh-http-deno/test/smoke.test.ts:293` — timeout in test
- `docs/principles.md:255` — principle: no sleep-based test timing

## Impact

Tests that rely on timing are inherently flaky and can produce false positives or negatives in CI environments with variable load.

## Remediation

1. Replace sleep-based synchronization with `notify`/`oneshot`/event-based coordination in Rust tests.
2. Replace `setTimeout` synchronization with proper async signalling in JS tests.

## Acceptance criteria

1. No sleep-based synchronization remains in critical test paths.
