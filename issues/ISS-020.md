# [ISS-020] `maxHeaderBytes = 0` / `null` disabling semantics are undocumented at runtime

## Summary

The server-limits documentation states that setting `maxHeaderBytes` to `0` or `null` disables the header size limit. The implementation feeds raw values directly to the parser with no special handling for zero, creating undefined behavior.

## Evidence

- `docs/features/server-limits.md:31` — states `0` or `null` disables the limit
- `crates/iroh-http-core/src/server.rs:573` — raw value fed into parser, no zero-handling

## Impact

Passing `maxHeaderBytes = 0` may cause unexpected behavior (or the panic described in ISS-001) rather than disabling the limit as documented.

## Remediation

1. Define explicit semantics for `maxHeaderBytes = 0` and `null` (disable vs. minimum clamp).
2. Implement the chosen logic in both parse-limit and post-parse enforcement.

## Acceptance criteria

1. The `0`/`null` behavior is deterministic, documented, and covered by a test.
