# [ISS-004] `maxRequestBodyBytes` does not explicitly reject with 413

## Summary

When the request body exceeds `maxRequestBodyBytes`, the body pump breaks out of the read loop without sending an explicit `413 Content Too Large` response. Documentation promises a 413 before body read completion.

## Evidence

- `crates/iroh-http-core/src/client.rs:336` — loop breaks on overflow with no explicit 413 response
- `docs/features/server-limits.md:20` — docs promise `413` before body read completes

## Impact

Callers do not receive the documented `413` status when they send oversized request bodies.

## Remediation

1. Convert body-limit overflow into an explicit error path.
2. Map that path to a `413` response (or update docs to match the actual chosen behavior).

## Acceptance criteria

1. An oversized request body test receives a deterministic `413` response.
