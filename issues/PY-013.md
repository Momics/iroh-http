# [PY-013] Async context manager returns detached node lifecycle state

## Summary

`IrohNode.__aenter__` creates and returns a new node object with a fresh close notifier, and `__aexit__` closes only the endpoint. This breaks closed-state consistency for code that awaits `node.closed` after context exit.

## Evidence

- `packages/iroh-http-py/src/lib.rs:976-984` — `__aenter__` returns a newly constructed `IrohNode`.
- `packages/iroh-http-py/src/lib.rs:982` — new node gets a fresh `close_notify`.
- `packages/iroh-http-py/src/lib.rs:995-997` — `__aexit__` closes endpoint but does not notify closed waiters.

## Impact

Users may observe inconsistent lifecycle behavior between `close()` and `async with` usage. Awaiting `node.closed` after context-manager exit can hang.

## Remediation

1. Return `self` (or equivalent shared state) from `__aenter__` instead of creating a detached replacement object.
2. Ensure `__aexit__` updates the same closed-state mechanism used by `close()`.
3. Add lifecycle parity tests covering explicit close and context-manager close paths.

## Acceptance criteria

1. `async with await create_node(...) as node:` preserves a single logical node lifecycle object.
2. After context exit, `await node.closed` resolves reliably.
3. Behavior of `close()` and `__aexit__` is consistent and documented.
