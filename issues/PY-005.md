# [PY-005] `compression_level` option is public but silently ignored

## Summary

`compression_level` is exposed in the Python API signature and type stubs but is explicitly unused in the Rust implementation.

## Evidence

- `packages/iroh-http-py/src/lib.rs:901` — parameter exposed
- `packages/iroh-http-py/iroh_http/__init__.pyi:16` — appeared in stubs
- `packages/iroh-http-py/src/lib.rs:922` — value is explicitly `_unused`

## Impact

Callers who pass `compression_level` believe they are configuring compression behavior, but the setting has no effect.

## Remediation

1. Either wire `compression_level` into core options, or remove/deprecate the parameter from the public API and stubs.
2. Add a behavior test for non-default compression configuration.

## Acceptance criteria

1. `compression_level` either has a documented and tested effect, or is absent from the public API.
