# [PY-009] Crypto tests contain no-op assertions

## Summary

The crypto test suite creates signatures but never calls or asserts `public_key_verify`. The tests pass unconditionally while providing no coverage of the verify path.

## Evidence

- `packages/iroh-http-py/tests/test_crypto.py:55` — signs but does not assert `verify`
- `packages/iroh-http-py/tests/test_crypto.py:75` — same pattern

## Impact

A broken `public_key_verify` implementation would go undetected by the existing test suite.

## Remediation

1. Replace the dead test setup with explicit positive and negative `verify` assertions.
2. Remove any setup code that does not contribute to assertions.

## Acceptance criteria

1. `test_crypto.py` has at least one positive and one negative `verify` assertion.
2. Introducing a bug in `public_key_verify` causes a test failure.
