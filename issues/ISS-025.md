# [ISS-025] Python `pyproject.toml` missing `[project.urls]`

## Summary

The `iroh-http-py` `pyproject.toml` has no `[project.urls]` section. This is required by the roadmap before publishing to PyPI.

## Evidence

- `packages/iroh-http-py/pyproject.toml:5` — no `[project.urls]` section
- `docs/roadmap.md:32` — roadmap requires `[project.urls]` with repository link

## Impact

The PyPI package page will have no repository link, reducing discoverability and trust.

## Remediation

1. Add a `[project.urls]` section with at minimum `Repository = "<url>"`.

## Acceptance criteria

1. PyPI metadata includes a repository URL.
