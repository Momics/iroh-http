# [ISS-014] Ticket format mismatch — docs claim Iroh standard format; code uses JSON

## Summary

The tickets documentation claims the library uses Iroh's standard base32/bech32 ticket format. The implementation serializes `NodeAddrInfo` as a JSON string for `node_ticket`.

## Evidence

- `docs/features/tickets.md:30` — claims Iroh standard base32/bech32 format
- `docs/features/tickets.md:25` — claims broader accepted peer formats
- `crates/iroh-http-core/src/lib.rs:244` — implementation uses JSON serialization for `node_ticket`
- `packages/iroh-http-shared/src/index.ts:58` — shared `ticketNodeId` parses JSON first
- `packages/iroh-http-shared/src/bridge.ts:457` — `fetch` accepts `PublicKey | string`, not arbitrary `NodeAddr`

## Impact

Interoperability expectations are incorrect. Consumers expecting standard Iroh ticket strings will receive JSON and vice versa.

## Remediation

1. Choose one ticket format contract (JSON or Iroh-native bech32) and enforce it consistently.
2. Update docs, helper APIs, and types to accurately reflect the chosen format.

## Acceptance criteria

1. Ticket format is unambiguous, documented with the exact encoding, and tested.
